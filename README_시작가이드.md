# 쌤버십 Flutter 앱 — 시작 가이드 (비개발자용)

> 이 문서는 **코딩을 모르는 분**이 이 앱을 컴퓨터에서 켜 보고, Cursor / Claude Code /
> Codex 같은 AI 도구로 이어서 완성해 나갈 수 있도록 단계별로 정리한 안내서입니다.
> 명령어는 **그대로 복사해서 붙여넣으면** 됩니다.

---

## 0. 이게 뭔가요? (3줄 요약)

- 기존 **웹(Next.js)** 과 **앱(Expo)** 을 참고해, 쌤버십을 **Flutter(안드로이드 우선)** 로 새로 만든 토대입니다.
- 가장 어렵고 차별화되는 기능(**연결노트 펜 필기**, **스캔 이미지 첨삭**, **요금제 결제 화면**, **질문방**)은 **이미 동작하도록** 만들어 두었습니다.
- 나머지 일반 화면(멘토찾기·커뮤니티·맞춤의뢰·마이페이지 등)은 **뼈대(스텁)** 만 있고, AI 도구로 채우면 됩니다. (→ 4단계)

## 1. 필요한 프로그램 딱 2개

| 프로그램 | 용도 | 설치 |
|---|---|---|
| **Flutter SDK** (최신 stable) | 앱을 빌드/실행 | https://docs.flutter.dev/get-started/install |
| **Android Studio** | 안드로이드 에뮬레이터 + 빌드 도구 | https://developer.android.com/studio |

> ⚠️ 이 앱은 비교적 **최신 Flutter(3.27 이상)** 기능을 씁니다. 설치할 때 "latest stable"
> 을 받으면 됩니다. 이미 깔려 있다면 터미널에서 `flutter upgrade` 한 번 실행하세요.

설치가 끝났는지 확인하려면 터미널(맥: 터미널, 윈도우: PowerShell)에 입력:

```bash
flutter doctor
```

체크 항목에 초록색 ✓ 가 뜨면 준비 완료입니다. 빨간 ✗ 가 있으면, 그 줄에 적힌
안내(예: "Android licenses")를 따라가면 됩니다. 막히면 그 화면을 캡처해서
Claude/Cursor에 "이거 어떻게 해결해?" 라고 물어보세요.

## 2. 앱 폴더 열고 패키지 받기

이 폴더(`ssambership_flutter`)를 **Cursor** 로 엽니다 (`File → Open Folder`).
그리고 Cursor 안의 터미널(`Terminal → New Terminal`)에 차례로 입력:

**① 먼저 안드로이드 실행 폴더를 만듭니다 (딱 한 번만):**

```bash
flutter create .
```

> ⚠️ **이 단계를 건너뛰면 앱이 안 켜집니다.**
> 이 프로젝트에는 앱의 본체 코드(`lib/`)만 들어 있고, 안드로이드를 실제로 구동하는
> 폴더(`android/`)는 일부러 빼 두었습니다(용량·환경마다 다름). 위 명령이 그 폴더를
> **여러분 컴퓨터에 맞게 자동 생성**해 줍니다. 마지막의 점(`.`)은 "지금 이 폴더에 만들어라"
> 라는 뜻이니 꼭 같이 적으세요. 기존 `lib/`·`pubspec.yaml` 은 덮어쓰지 않으니 안심해도 됩니다.

**② 그다음 부품(패키지)을 받습니다:**

```bash
flutter pub get
```

- 이 명령은 앱이 쓰는 부품(패키지)들을 인터넷에서 받아옵니다. 1~3분 걸립니다.
- 혹시 빨간 에러로 멈추면, 그 메시지를 **그대로 복사**해서 Claude Code/Cursor에
  "pub get 중 이 에러가 났어, 고쳐줘" 라고 붙여넣으세요. (대개 버전 한두 줄 조정이면 끝납니다.)

> 📷 카메라로 종이를 스캔하는 기능(`cunning_document_scanner`)이나 갤러리 접근은
> **카메라·사진 권한**이 필요합니다. `flutter create .` 가 만든 `android/app/src/main/AndroidManifest.xml`
> 에 권한 한두 줄을 추가해야 할 수 있습니다. 막히면 AI 도구에 "안드로이드 카메라/갤러리 권한을
> 매니페스트에 추가해줘" 라고 요청하세요.

## 3. 안드로이드에서 켜 보기 (가장 빠른 확인)

1. **Android Studio** 를 켜고 → `Device Manager` → 에뮬레이터(가상 휴대폰) 하나를 실행합니다.
   (태블릿 UI도 보고 싶으면 "Pixel Tablet" 같은 태블릿 기기를 추가로 만들면 좋습니다.)
2. 에뮬레이터가 켜진 상태에서, 터미널에 입력:

```bash
flutter run
```

3. 잠시 뒤 앱이 에뮬레이터에 뜹니다. **진입 화면에서 할 수 있는 것:**
   - **"학생으로 시작"** → 질문방 목록 → 방 하나 탭 → **연결노트(필기)** / **스캔 첨삭** 진입
   - **"멘토로 시작"** → 멘토 질문방 → 같은 방식으로 첨삭 기능 확인
   - 아래쪽 **"연결노트 필기 미리보기 / 스캔 첨삭 미리보기"** 버튼으로 핵심 기능에 **바로** 진입

> 💡 펜 필기는 마우스로도 그려집니다. 실제 **스타일러스(펜) + 태블릿** 에서는 손바닥이
> 닿아도 무시(팜 리젝션)되고, 손가락으로는 확대/이동이 됩니다.

실행 중 코드를 고치면 터미널에서 **`r`** 키(핫 리로드)로 즉시 반영, **`R`** 로 재시작,
**`q`** 로 종료할 수 있습니다.

## 4. AI 도구로 "이어서" 완성하는 법 (핵심)

이 프로젝트는 **AI가 이어 작업하기 좋게** 설계돼 있습니다. 빈 화면(스텁)에는 안내 문구와
"무엇을 채워야 하는지"가 적혀 있고, 참고할 웹/Expo 위치도 표시돼 있습니다.

### 추천 워크플로우

1. **무엇을 만들지 한 줄로 정합니다.** 예: "학생 커뮤니티 홈 화면을 채워줘."
2. **Claude Code(또는 Cursor)** 에게 아래처럼 부탁합니다. 참고 문서를 같이 가리키면 품질이 확 올라갑니다:

   > "`docs/EXPO_FEATURE_REFERENCE.md` 의 커뮤니티 항목과 `docs/ARCHITECTURE.md` 의
   > 디자인 토큰을 참고해서, `lib/features/student/screens/student_screens.dart` 의
   > `CommunityHomeScreen` 스텁을 실제 화면으로 만들어줘. 색/여백/카드 스타일은
   > 이미 있는 `StudentRoomListScreen` 과 똑같은 톤으로 맞춰줘."

3. AI가 코드를 고치면, 터미널에서 `r`(핫 리로드)로 바로 확인합니다.
4. 데이터(목록·글쓰기 등)가 필요하면 **5단계(Supabase 연결)** 를 먼저 합니다.

### 각 도구를 언제 쓰면 좋은가 (참고)
- **Claude Code / Codex (터미널형)**: "이 파일 채워줘", "이 에러 고쳐줘" 처럼 **여러 파일을 한 번에** 다룰 때 강력합니다.
- **Cursor (에디터형)**: 코드를 **보면서** 부분 수정하거나, 한 화면을 다듬을 때 편합니다.
- **ChatGPT**: 막힌 개념 질문, 카피 문구·아이콘 아이디어 등 **대화형 보조**.

> ✅ 황금률: **이미 완성된 화면을 "본보기"로 가리키세요.** ("RoomList처럼 만들어줘")
> 그러면 디자인이 들쭉날쭉해지지 않습니다.

## 5. 실제 데이터 연결하기 (Supabase) — 선택

지금은 키 없이도 화면이 보이도록 해두었습니다(데모 모드). **진짜 로그인·질문방·결제**
데이터를 붙이려면, 웹과 **같은 Supabase** 프로젝트의 키 2개가 필요합니다.

웹 코드(`ssambership_web`)의 환경설정(`.env`)에서 아래 두 값을 찾습니다:
- `NEXT_PUBLIC_SUPABASE_URL`
- `NEXT_PUBLIC_SUPABASE_ANON_KEY`

그리고 앱을 실행할 때 **키를 주입**합니다(코드에 직접 쓰지 마세요):

```bash
flutter run \
  --dart-define=SUPABASE_URL=여기에_URL \
  --dart-define=SUPABASE_ANON_KEY=여기에_ANON_KEY
```

> 🔒 보안: 키를 소스코드 파일에 적어 깃허브에 올리면 위험합니다. 위처럼 실행 시점에
> 넣는 방식이 안전합니다. (`lib/core/supabase/supabase_client.dart` 가 이 값을 읽습니다.)

연결 뒤에는 AI에게 "Supabase의 `mentor_student_rooms` 테이블에서 내 질문방 목록을
가져와 `StudentRoomListScreen` 에 실제로 표시해줘" 처럼 부탁하면 됩니다. 테이블 구조는
`docs/ARCHITECTURE.md` 의 "데이터 모델" 절에 정리돼 있습니다.

## 6. 지금 무엇이 되어 있고, 무엇이 남았나

### ✅ 이미 동작 (그대로 써도 됨)
- **진입/역할 선택** 화면
- **연결노트 펜 필기 에디터** — 펜/형광펜/지우개, 굵기, 색, 실행취소/다시실행, 팜 리젝션,
  확대·이동, 종이 템플릿(무지/줄/모눈/점선), 자동저장, 텍스트+필기 하이브리드
- **스캔 이미지 첨삭 에디터** — 스캔/갤러리 → 이미지 위 펜 첨삭, **좌표 정규화(0~1)** 로
  어떤 기기·확대에서도 위치가 어긋나지 않음, 작성자(멘토/학생) 색 구분, 원본만 보기
- **요금제/캐시 결제** 화면 — limited / standard / premium (가격·혜택 잠금값 반영)
- **질문방** 목록·상세 (학생/멘토) — 위 두 첨삭 기능으로 들어가는 진입점
- **반응형 셸** — 폰은 하단 탭, **태블릿은 측면 레일**로 자동 전환

### 🚧 뼈대만 있음 (AI로 채우기 — 4단계 방식)
- 멘토 찾기 / 멘토 상세
- 커뮤니티 홈 · 게시글 · 숏폼
- 맞춤의뢰(학생/멘토) 흐름
- 마이페이지 / 멘토 프로필 / 멘토 대시보드 · 정산

각 빈 화면을 누르면 "무엇을 만들면 되는지 + 참고할 웹/Expo 위치"가 화면에 적혀 있습니다.

### 다음 한 걸음 추천 순서
1. **Supabase 연결**(5단계) → 질문방 목록을 실제 데이터로
2. **커뮤니티 홈** 채우기 (사용 빈도 높음)
3. **마이페이지/프로필**
4. **맞춤의뢰** 흐름
5. 연결노트 저장본을 **Supabase Storage 업로드**까지 연결 (지금은 저장 직전까지 구현, 업로드 자리만 비움)

## 7. 자주 만나는 문제 (빠른 처방)

| 증상 | 처방 |
|---|---|
| `flutter: command not found` | Flutter 설치 후 PATH 설정 필요. `flutter doctor`가 되는 터미널에서 작업하세요. |
| `pub get` 빨간 에러 | 에러 메시지를 그대로 Claude Code에 붙여넣고 "고쳐줘". 보통 `pubspec.yaml` 버전 한두 줄 조정. |
| 에뮬레이터에 앱이 안 뜸 | Android Studio에서 기기가 "실행 중"인지 확인 → `flutter run` 다시. |
| 빌드 중 SDK 버전 에러 | `flutter upgrade` 로 최신 stable로 올리세요(이 앱은 3.27+ 필요). |
| 스캔/카메라가 에뮬레이터에서 안 됨 | 카메라 스캔은 **실기기**에서 동작. 에뮬레이터에서는 "갤러리에서 선택"으로 테스트하세요. |
| 화면이 깨져 보임 | 어떤 화면인지 캡처해서 "이 화면 레이아웃 고쳐줘 (태블릿/폰)" 라고 AI에 요청. |

## 8. 폴더 지도 (어디에 뭐가 있나)

```
ssambership_flutter/
├─ README_시작가이드.md      ← (지금 이 문서)
├─ pubspec.yaml             ← 앱이 쓰는 패키지 목록 (버전)
├─ docs/
│  ├─ ARCHITECTURE.md          ← 앱 구조·디자인 토큰·데이터 모델 설명
│  ├─ EXPO_FEATURE_REFERENCE.md ← Expo 화면 ↔ 웹 경로 ↔ Flutter 화면 매핑표
│  └─ QA_PLAN.md               ← 테스트 체크리스트 + 알려진 위험
└─ lib/
   ├─ main.dart / app.dart        ← 앱 시작점
   ├─ core/                       ← 공통: 색/테마/반응형, 라우팅, 모델, Supabase
   └─ features/                   ← 기능별 화면
      ├─ handwriting/             ← ★ 펜 필기 엔진 (연결노트·스캔이 공유)
      ├─ connection_note/         ← ★ 연결노트 필기 에디터
      ├─ scan_annotation/         ← ★ 스캔 이미지 첨삭 에디터
      ├─ student/ · mentor/       ← 학생·멘토 화면 (일부 스텁)
      ├─ community/ commission/   ← 커뮤니티·맞춤의뢰 (스텁)
      ├─ cash/                    ← 요금제·결제
      └─ shell/                   ← 반응형 네비게이션 셸
```

막히는 부분이 있으면 언제든 그 화면/에러를 캡처해서 AI 도구에 물어보세요.
이 토대 위에서 **"본보기 화면을 가리키며" 하나씩** 채워가면 됩니다. 화이팅!
