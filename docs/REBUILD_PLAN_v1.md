# 쌤버십 앱 — 코드 QA + 첫 화면부터 UI/UX 재구성 기획서

> 작성 기준: **웹앱(`ssambership_web-main`)이 디자인·기능의 기준점**, 연결노트 필기/스캔 첨삭은 기획서 우선.
> 이 문서는 두 부분입니다. **(1) 어제 만든 앱 코드의 QA 결과**, **(2) 웹을 기준으로 첫 화면부터 다시 짜는 화면별 기획.**
> 화면 상태 표기: ✅ 실제 동작 · 🟡 뼈대만(스텁) · ⛔ 아직 없음(신규 필요)

---

# Part 0. 코드 QA 결과 (어제 만든 앱)

## 0-1. 한 줄 요약

앱의 **토대와 차별화 기능(연결노트 필기 · 스캔 첨삭)은 실제로 구현되어 동작**하고, 표준 화면(커뮤니티·맞춤의뢰·마이페이지 등)은 **의도적으로 뼈대만** 둔 상태입니다. 빌드를 막던 코드 오류는 **단 2개였고 이미 해결**했습니다.

## 0-2. 발견된 버그와 처리

| 심각도 | 내용 | 위치 | 상태 |
|---|---|---|---|
| **Critical** | `PointerDeviceKind` import 누락 (펜/손가락 구분용) | `handwriting_controller.dart` | ✅ `gestures.dart` 추가로 해결 |
| **Critical** | `GlobalKey` import 누락 (화면 캡처용) | `canvas_exporter.dart` | ✅ `widgets.dart` 추가로 해결 |

전체 28개 파일을 정적 스캔(심볼↔import 정합 검사)한 결과, **위 2개 외 추가 컴파일 오류는 없음**을 확인했습니다. (마지막 빌드가 7분간 정상 진행되다 이 2개에서만 멈췄던 것과 일치합니다.)

## 0-3. 점검했고 "이상 없음"으로 확인된 부분

- **스캔 첨삭 좌표 매퍼**(`ScanCoordMapper`) — 가장 버그 위험이 높다고 봤던 부분인데, `Offset/Size`에 의존하지 않고 **순수 비율(double)로 정규화**하도록 짜여 있어 구조적으로 안전합니다. 확대/기기 무관 정합의 토대가 잘 잡혀 있습니다.
- **모델 파일들**(`note`/`user`/`ink_stroke`/`ink_sketch`) — import 누락 없음.
- **세션 관리**(`session.dart`) — `ChangeNotifier`가 `foundation.dart`로 정상 공급됨(오탐이었음).

## 0-4. 화면 구현 상태 인벤토리 (실측)

| 영역 | 화면 | 상태 |
|---|---|---|
| 인증 | 런치/역할선택 (`LaunchScreen`) | ✅ |
| 질문방(학생) | 방 목록 · 방 상세 | ✅ |
| 질문방(멘토) | 방 목록 · 방 상세 | ✅ |
| **연결노트** | 필기 에디터 (펜·지우개·undo/redo·자동저장) | ✅ 차별화 |
| **스캔 첨삭** | 스캔 진입 · 주석 에디터 (정규화 좌표) | ✅ 차별화 |
| 캐시 | 요금제 선택/결제 UI (`CashScreen`) | ✅(UI), 결제연동 ⛔ |
| 멘토 | 대시보드 · 정산 · 프로필 · 맞춤의뢰 | 🟡 스텁 |
| 학생 | 멘토검색 · 커뮤니티 · 맞춤의뢰 · 마이페이지 | 🟡 스텁 |
| 공통 | 알림 | ⛔ 없음 |

## 0-5. 실행 환경 관련(코드 외) 알려진 사항

- 카메라 문서 스캔(`cunning_document_scanner`)은 실기기 전용이라 **갤러리 불러오기로 먼저 동작** 확인 권장. 빌드 중 뜬 관련 노란 경고(KGP)는 **무시 가능**(미래 버전 예고일 뿐 빌드를 막지 않음).
- 결제(토스페이먼츠) 앱 연동, Supabase 실데이터 연결은 **미연동**(현재 더미 데이터).

# Part 1. 설계 기준 (웹 = 단일 기준점)

## 1-1. 잠금 디자인 토큰 (변경 금지)

| 토큰 | 값 | 용도 |
|---|---|---|
| Primary | `#1A56DB` | 주요 버튼, 강조, 링크 |
| Secondary | `#3F83F8` | 보조 강조 |
| Accent | `#F59E0B` | 별점, 포인트 |
| Success | `#10B981` | 완료/긍정 |
| Danger | `#EF4444` | 경고/삭제 |
| Background | `#F9FAFB` | 화면 배경 |

카드: 흰 배경 + 큰 모서리 둥글기(16px 안팎) + 옅은 테두리/그림자, 여백을 넉넉히. 친근한 한국어 카피("~해요" 체).

## 1-2. 요금제 (잠금 — 목업으로 재확인됨)

| 요금제 | 색 | 가격(월) | 질문 한도 | 비고 |
|---|---|---|---|---|
| **Limited** | 초록(새싹) | 55,000캐시 | 주 4개 | 가볍게 시작 |
| **Standard** | 파랑(왕관) | 114,900캐시 | 주 9개 | **추천** 배지 |
| **Premium** | 보라(다이아) | 249,900캐시 | 무제한 | 완전 관리 |

공통: "연결노트 제공 · 질문 히스토리 무제한", 가격은 **(VAT 포함)** 표기. **무료 질문은 한 멘토당 최대 3개.** 요금제는 **멘토 1명 기준**(다른 멘토 구독 시 별도 한도).

## 1-3. 캐시·수수료 규칙

- 1캐시 = 1원. 잔액은 `balance_cents ÷ 100`.
- 수수료: **구독 30/70**(플랫폼/멘토), **맞춤의뢰 20/80**.

## 1-4. 역할

**학생 · 멘토 · 관리자** 3종. 앱은 학생/멘토 중심(관리자는 웹 콘솔). 로그인 후 역할에 맞는 홈으로 분기.

## 1-5. 목업에서 관찰된 디자인 언어 (앱 적용 지침)

- **공지 바**: 화면 최상단 얇은 띠(프로모션/공지) — 앱에선 홈 상단 카드로.
- **멘토 카드**: 아바타 + 인증 체크(✓) + 학교/학과/학번 + 과목 태그(칩) + 별점(주황) + 답변률.
- **단계 표시기(stepper)**: 맞춤의뢰처럼 다단계 흐름은 1·2·3·4 진행바로 현재 위치를 명확히.
- **상태 배지**: NEW, 추천, 우수/프리미엄 멘토 등급 등 작은 컬러 칩.
- **우측 안내 레일**(웹): 마감 안내/이용 안내 — 앱(모바일)에선 본문 하단 접이식 카드로, 태블릿에선 2단 레이아웃 우측에 유지.

# Part 3. 첫 화면부터 화면별 재구성 기획

각 화면을 **흐름 순서**로, 〔목적 → 웹 기준 UI → UX/흐름 → 현재 상태 → 앱 재구성 제안(모바일/태블릿·개선점)〕 형식으로 정리합니다.

---

## 3-1. 스플래시 → 게스트 홈 (첫 화면)

**목적** — 앱을 켜면 처음 보이는 화면. 로그인 전에도 둘러볼 수 있는 게스트 메인.

**웹 기준 UI** — 상단 공지 바("6월 모의고사 기념 1주 무료 + 무료 15질문") → 헤더(로고 + 멘토찾기/질문방/커뮤니티/맞춤의뢰/캐시결제 + 로그인/회원가입) → 히어로("공부는 혼자, 성장은 함께" + 두 CTA: 멘토 찾기 / 무료 체험 시작하기 + 떠다니는 알림 카드 3개) → 지표 4종(전국 멘토 2,500+ / 누적 질문 120,000+ / 만족도 4.9 / 전 과목) → 기능 카드 4종(1:1 맞춤 / 무제한 질문 / 연결노트 / 안전 결제) → 추천 멘토 가로 캐러셀 → 하단 무료체험 배너.

**UX/흐름** — 로고 클릭·직접 접속 모두 동일한 게스트 홈. **로그인 직후에만** 역할 홈으로 이동.

**현재 상태** — 🟡 런치 화면(역할 선택 + 데모 버튼)만 있음. 게스트 메인은 ⛔.

**앱 재구성 제안**
- 스플래시(로고 1초) → 게스트 홈. 게스트 홈을 **앱의 진짜 첫 화면**으로 신설(현재 런치 화면을 이걸로 승격/대체).
- 모바일: 히어로 → 지표(2×2) → 기능 카드(세로) → 추천 멘토(가로 스크롤) → 무료체험 버튼(하단 고정).
- 태블릿: 히어로를 2단(좌 텍스트/우 이미지), 추천 멘토 한 줄에 3~4장.
- 상단 공지 바는 **닫기 가능한 카드**로(웹의 X 버튼 동일).
- 하단 탭: 홈 / 멘토 / 질문방 / 커뮤니티 / 내정보 (게스트는 보호된 탭 진입 시 로그인 유도).
- **개선점**: 게스트가 "무료 체험 시작하기"를 누르면 곧장 가입 흐름으로, 가입 후 첫 멘토 구독까지 자연스럽게 잇는 온보딩 동선 추가.

---

## 3-2. 회원가입 / 로그인 (역할)

**목적** — 계정 생성과 로그인. 학생/멘토 분리.

**웹 기준 UI** — `/signup`(통합 가입), `/login/student`·`/login/mentor`(역할별 로그인). 결제·구독 화면에 "로그인이 필요해요 → 로그인하기 / 회원가입하기" 유도 카드.

**UX/흐름** — 보호된 행동(구독/질문/의뢰)을 시도하면 로그인으로 보냈다가, 끝나면 원래 위치로 복귀(웹 미들웨어의 `next` 복귀 경로 개념).

**현재 상태** — 🟡 데모용 역할 전환만. 실제 인증 ⛔.

**앱 재구성 제안**
- 가입: 이메일 → 프로필(이름/닉네임/학년 또는 멘토 학교·학과·인증) 2단계. 미성년자 동의(웹 `legal/minor-consent`)를 가입 단계에 포함.
- 로그인: 역할 선택 후 이메일 로그인 + 소셜(선택). 비밀번호 찾기(`forgot-password`).
- Supabase Auth 연동: 로그인 성공 시 `session`을 실제 사용자로 교체(현재 `DemoSession` 자리).
- **개선점**: "보호된 화면 진입 → 로그인 → 원래 화면 복귀"를 앱 라우터에도 구현(현재는 역할 redirect만 있음).

## 3-3. [학생] 홈 대시보드

**목적** — 로그인한 학생의 출발점. 내 구독 멘토, 최근 질문, 캐시 잔액 한눈에.

**웹 기준 UI** — `(student)/home`. 로그인 헤더(알림 벨 + 메시지 배지 + 프로필). 게스트 홈과 달리 **내 활동 중심**(내 멘토/방, 캐시 잔액, 알림).

**UX/흐름** — 홈 → 각 질문방/연결노트/캐시로 빠르게 진입.

**현재 상태** — 🟡(학생 셸·방 목록은 있으나 종합 홈은 ⛔).

**앱 재구성 제안**
- 상단: 인사 + 캐시 잔액 카드(충전 버튼). 
- 섹션: 내 구독 멘토(가로 카드) → 진행 중 질문방(최근순) → 새 답변/연결노트 업데이트 알림 → 추천 멘토.
- 태블릿: 좌측 레일(탭) + 본문 2단(좌 질문방 리스트 / 우 선택 상세) 구도까지 확장 가능.

---

## 3-4. [학생] 멘토 찾기 → 멘토 상세 → 구독 요금제

**목적** — 멘토 탐색하고 구독으로 연결.

**웹 기준 UI**
- 멘토 찾기(`/mentors`): 멘토 카드 그리드(아바타·인증·학교/학과·과목 태그·별점·답변률), 검색/필터.
- 멘토 상세(`/mentors/[id]`): 프로필 + 소개 + 후기 + "구독하기".
- 구독 요금제(`subscribe`): **목업 확인** — 멘토 헤더 카드 + 3요금제 카드(Limited 초록/Standard 파랑·추천/Premium 보라) + 우측 "선택 요약"(선택 요금제·월 구독 요금·결제 금액 VAT 포함) + 로그인 유도 + 하단 혜택 4종 + 안전 결제 안내. 안내: "각 요금제는 한 멘토 기준 · 무료 질문 최대 3개".

**UX/흐름** — 멘토 찾기 → 상세 → 요금제 선택 → (로그인) → 결제 → 질문방 개설.

**현재 상태** — 멘토 검색 🟡 / 요금제 UI는 `CashScreen`에 근접 ✅(잠금 가격 반영) / 결제 연동 ⛔.

**앱 재구성 제안**
- 멘토 찾기: 상단 검색 + 과목 필터 칩 + 카드 리스트(모바일 1열 / 태블릿 2~3열).
- 멘토 상세: 헤더 카드 → 과목·소개 → 후기 → 하단 고정 "구독하기".
- 요금제: 3카드 세로(모바일), 가로 3열(태블릿). Standard에 추천 배지·파랑 강조. **결제 직전 "선택 요약" 카드**(가격·VAT) 재현.
- **개선점**: 무료 질문 잔여(한 멘토당 3개)를 요금제 화면과 질문 작성 화면에 **항상 표시**해 혼동 방지.

## 3-5. [학생] 질문방 → 방 상세 → 질문 스레드

**목적** — 구독 멘토에게 질문하고 답변받는 핵심 활동.

**웹 기준 UI** — `question-room`(방 목록) → `[roomId]`(방 상세, 스레드 목록) → `thread/[threadId]`(질문/답변 대화). 데이터: 방→스레드→메시지 + 첨부.

**UX/흐름** — 방 선택 → 스레드 목록 → 스레드 열기 → 질문 작성(텍스트+이미지/첨부) → 멘토 답변 → 연결노트/스캔 첨삭으로 이어짐.

**현재 상태** — 방 목록·상세 ✅ / 스레드 채팅 화면 🟡.

**앱 재구성 제안**
- 방 목록: 멘토별 카드(최근 메시지·미읽음 배지).
- 방 상세: 상단 멘토 요약 + 액션(질문하기 / 연결노트 / 스캔 첨삭) + 스레드 목록.
- 스레드: 채팅형 UI(질문=학생 색, 답변=멘토 색), 첨부 썸네일, 하단 입력바(텍스트+사진+스캔).
- 태블릿: 좌 스레드 목록 / 우 대화 2단.
- **개선점**: 질문 작성 시 무료/유료 차감을 미리 안내(요금제 한도 연동).

## 3-6. ★ 연결노트 (필기) — 차별화 기능

**목적** — 멘토·학생이 함께 쓰는 공개형 학습 노트. 키보드 텍스트를 넘어 **펜 필기 + 텍스트 하이브리드**(굿노트/노타빌리티 지향).

**기준** — 기획서 우선. 방(room) 단위·공개형·RLS. 카테고리(전체/멘토에게 요청/멘토가 요청/메모).

**UX/흐름** — 방 상세 → 연결노트 → 펜/형광펜/지우개로 필기 + 텍스트 박스 + 이미지 주석 → 자동 저장(벡터 JSON + 썸네일 PNG).

**현재 상태** — ✅ 구현됨. 펜·형광펜·지우개·굵기·undo/redo·팜 리젝션·줌/팬·종이 템플릿·자동저장.

**앱 재구성 제안 / 개선점**
- 태블릿 우선: 측면 도구 레일(펜·색·굵기), 스타일러스 팜 리젝션 강조.
- 카테고리 탭(전체/요청/메모)을 노트 상단에 노출.
- **개선점**: 저장 상태 표시("저장됨"), 버전/되돌리기 안정화(QA의 자동저장 항목과 연동). 손가락 전용 기기에선 펜 전용 토글 안내.

## 3-7. ★ 스캔 이미지 첨삭 — 차별화 기능

**목적** — 종이 문제/답안을 스캔(또는 사진)해 그 위에 멘토가 첨삭. **이미지 기준 정규화 좌표**로 확대·기기 무관 정합.

**기준** — 기획서 우선. 스캔=배경(편집 불가) + 펜=주석(벡터) 레이어. 저장: 원본 + 주석 JSON + 미리보기 PNG. 작성자(멘토/학생) 색 구분.

**UX/흐름** — 방 상세 → 스캔/첨삭 → (카메라 스캔 또는 갤러리) → 이미지 위 펜 주석 → 저장.

**현재 상태** — ✅ 구현됨. 좌표 매퍼(`ScanCoordMapper`)가 비율 기반이라 구조 안전(QA 확인). 카메라 스캔은 실기기 전용이라 갤러리로 먼저 동작.

**앱 재구성 제안 / 개선점**
- 진입 화면에 "카메라로 스캔 / 갤러리에서 선택" 2버튼.
- 첨삭 화면: 배경 이미지 고정 + 주석 레이어 토글 + 작성자 색 자동.
- **개선점(QA 최우선 테스트와 연동)**: 확대 후/재실행 후 주석 위치 유지를 실기기에서 반드시 검증. 세로·가로 사진 모두 정합 확인.

## 3-8. [학생] 캐시 지갑 · 충전 · 내역

**웹 기준 UI** — `wallet`(잔액) · `wallet/charge`(충전, 성공/실패 분기) · `wallet/ledger`(내역) · `cash-history`.

**UX/흐름** — 충전 → 토스페이먼츠 결제 → 성공 시 잔액 반영 → 구독/의뢰에 사용.

## 3-9. [학생] 맞춤의뢰 (게시 → 지원 대기 → 멘토 선택 → 진행방)

**목적** — 정형 구독 외, 특정 과제(예: 발표 피드백)를 의뢰하고 멘토 지원을 받아 1명 선택.

**웹 기준 UI (목업 확인)** — 4단계 stepper(요청 작성 → 정보 확인 → **멘토 지원 대기** → 멘토 선택). 지원 대기 화면: 요청 요약(제목·카테고리·예산·마감) + 지원 현황(지원자 수·카운트다운·진행바) + 최근 지원 멘토 카드(NEW 배지, 연락처는 선택 후 공개) + 우측 마감/이용 안내. 이후 선택 → 진행방 → 납품/검수.

**UX/흐름** — 작성 → 지원 모집 → 비교 후 1명 선택 → 진행방에서 작업·납품 → 완료/검수. 수수료 20/80.

**현재 상태** — 🟡 전부 스텁.

**앱 재구성 제안**
- 작성: 제목·카테고리·예산(캐시)·마감일·상세 + 첨부.
- 지원 대기: stepper(모바일=가로 컴팩트) + 지원자 카드 리스트 + 카운트다운 + "요청 수정/계속 기다리기".
- 진행방: 의뢰 전용 채팅 + 납품물 업로드/다운로드 + 검수·수정 요청.
- 태블릿: 지원자 비교를 2열 카드로.
- **개선점**: "선택 전 연락처 비공개" 규칙을 화면에 명시(웹과 동일), 안전거래 안내·신고 동선 포함.

# Part 4. 기획 공백 보완 (재량 개선 제안)

웹/기획을 앱으로 옮길 때 비어 있거나 약한 부분을 앱 맥락에 맞게 보완 제안합니다.

1. **온보딩 동선** — 게스트 홈 → "무료 체험" → 가입 → 첫 멘토 구독까지 한 흐름으로 연결(웹은 페이지 단위라 끊김).
2. **보호 라우팅 복귀** — 보호 화면 진입 시 로그인 후 원래 위치 복귀(웹 미들웨어 개념을 앱 라우터에 이식).
3. **무료 질문 잔여 표시** — 한 멘토당 3개를 요금제·질문 작성·방 상세에 일관 노출.
4. **저장 상태 가시화** — 연결노트·스캔 첨삭에 "저장됨/저장 중" 표시(QA 자동저장 리스크 완화).
5. **태블릿 2단 레이아웃** — 질문방·맞춤의뢰·연결노트에서 목록/상세 2단(태블릿 주 사용 환경 반영).
6. **안전·정책 노출** — 미성년 동의, 환불, 커뮤니티 가이드, 신고 동선을 관련 화면에 인라인 링크.
7. **결제/출금 보안** — 카드·계좌 정보를 앱에서 직접 입력받지 않고 검증된 결제창/흐름으로 위임.

# Part 5. 우선순위 로드맵 (재구성 순서)

빌드가 살아난 지금, 아래 순서로 채우면 가장 효율적입니다.

1. **게스트 홈 + 가입/로그인(실 인증)** — 첫 화면과 진입을 진짜로 완성(현재 런치/데모 → 정식).
2. **학생 홈 + 질문방 스레드(채팅)** — 핵심 사용 루프.
3. **차별화 기능 실기기 QA 마감** — 연결노트 자동저장 + 스캔 좌표 정합(태블릿/스타일러스).
