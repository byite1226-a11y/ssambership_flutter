# 쌤버십 Flutter 앱 — 1차 QA 계획서 (Task 5)

이 문서는 **앱을 처음 돌려보면서 무엇을 어떤 순서로 점검할지**, 그리고 버그가 나왔을 때 **무엇부터 고칠지**를 정리한 체크리스트입니다.
코딩을 직접 하지 않아도, 이 순서대로 따라가며 "되는 것 / 안 되는 것"을 표시한 뒤, 그 표를 그대로 Cursor·Claude Code·Codex 같은 AI 도구에 붙여 넣으면 됩니다.

> 핵심 원칙: **위(Critical)부터 아래(Low)로** 잡습니다. 앱이 아예 안 켜지는 문제를 두고 색깔 같은 사소한 걸 먼저 만지면 시간을 크게 낭비합니다.

---

## 0. QA를 시작하기 전에

1. `README_시작가이드.md` 의 순서대로 `flutter pub get` 까지 끝났는지 확인합니다.
2. 안드로이드 폰(또는 에뮬레이터) **그리고** 태블릿(또는 태블릿 에뮬레이터) 두 종류를 준비하면 가장 좋습니다. 태블릿이 이 앱의 주요 사용 환경이기 때문입니다.
3. 점검은 **3단계**로 나눠서 합니다.
   - **A. 빌드 단계** — 컴파일이 되는가 (코드가 말이 되는가)
   - **B. 부팅·이동 단계** — 앱이 켜지고 화면 간 이동이 되는가
   - **C. 기능 단계** — 각 화면이 실제로 동작하는가

## 1. 심각도(Severity) 정의

| 등급 | 의미 | 처리 |
|------|------|------|
| **Critical (치명)** | 앱이 빌드 안 됨 / 켜자마자 꺼짐 / 핵심 기능 자체가 막힘 | **즉시.** 이게 살아야 나머지 점검 가능 |
| **High (높음)** | 특정 화면이 깨지거나 데이터가 저장/표시 안 됨. 우회 불가 | 그날 안에 |
| **Medium (보통)** | 동작은 하나 결과가 이상함, 레이아웃 깨짐, 일부 기기에서만 문제 | 기능 완성 후 정리 |
| **Low (낮음)** | 색·간격·문구·정렬 등 미관. 사용엔 지장 없음 | 마지막에 한꺼번에 |

## 2. A단계 — 빌드 점검 (Critical 영역)

> 목표: `flutter run` 이 에러 없이 통과.

- [ ] **C-1.** `flutter pub get` 시 의존성 충돌 메시지가 없는가
  - 위험 포인트: `scribble`, `perfect_freehand`, `cunning_document_scanner` 의 버전. pub.dev가 더 최신을 요구하면 버전 숫자를 맞춰야 함.
  - 증상 예: `version solving failed`
- [ ] **C-2.** `flutter run` 시 빨간 컴파일 에러가 없는가
  - 가장 가능성 있는 곳: **`handwriting_painter.dart`** 의 `perfect_freehand` 호출부. (정적 검증은 끝냈지만, pub get으로 실제 버전이 깔린 뒤에만 100% 확인 가능)
  - 만약 `getStroke` / `StrokeOptions` / `PointVector` 관련 에러가 뜨면 → AI 도구에 "현재 설치된 perfect_freehand 버전의 API에 맞춰 이 파일을 고쳐줘" 라고 요청.
- [ ] **C-3.** 안드로이드 최소 SDK / Gradle 관련 에러가 없는가
  - `cunning_document_scanner`, `image_picker` 는 카메라·저장소 권한이 필요. `AndroidManifest.xml` 에 권한 추가가 필요할 수 있음.

**A단계가 안 끝나면 B로 가지 마세요.** 빌드부터 살리는 게 1순위입니다.

## 3. B단계 — 부팅·화면이동 점검

> 목표: 앱이 켜지고, 역할 선택 후 탭들 사이를 오갈 수 있다.

- [ ] **B-1.** 앱 실행 시 첫 화면(역할 선택 / 데모 버튼)이 뜨는가 — *Critical*
  - 안 뜨고 흰 화면/즉시 종료면: `main.dart` 의 Supabase 초기화에서 막혔을 가능성. (키 없이도 켜지도록 안전장치를 넣어뒀지만, 실제 환경에서 재확인)
- [ ] **B-2.** "학생으로 시작" → 질문방 목록 화면으로 이동하는가 — *Critical*
- [ ] **B-3.** "멘토로 시작" → 멘토 질문방 목록으로 이동하는가 — *Critical*
- [ ] **B-4.** 하단 탭(폰) / 좌측 레일(태블릿) 5개가 모두 눌리고 화면이 바뀌는가 — *High*
- [ ] **B-5.** 질문방 상세 → "연결노트" 버튼 → 필기 화면으로 들어가는가 — *High*
- [ ] **B-6.** 질문방 상세 → "스캔/첨삭" 버튼 → 스캔 진입 화면으로 들어가는가 — *High*
- [ ] **B-7.** 뒤로가기를 눌렀을 때 앱이 죽지 않고 이전 화면으로 잘 돌아오는가 — *Medium*
- [ ] **B-8.** 데모 버튼(연결노트 데모 / 스캔 첨삭 데모)이 로그인 없이도 열리는가 — *Medium*

## 4. C단계 — 차별화 기능 집중 점검 ★가장 중요★

이 앱의 핵심이자 **가장 버그가 날 만한 곳**입니다. 여기에 시간을 가장 많이 쓰세요.

### 4-1. 연결노트 필기 (handwriting)

- [ ] **펜 그리기** — 손가락/펜으로 그었을 때 선이 그려지는가 — *Critical*
- [ ] **굵기 변화** — 천천히/빠르게 그을 때 선 굵기가 자연스럽게 변하는가 (perfect_freehand 압력 흉내) — *Medium*
- [ ] **지우개** — 획 단위로 지워지는가 — *High*
- [ ] **실행취소(undo) / 다시실행(redo)** — 버튼이 동작하는가 — *High*
- [ ] **전체 지우기** — 확인 후 모두 지워지는가 — *Medium*
- [ ] **펜 색 / 형광펜** — 색이 바뀌고, 형광펜이 반투명으로 겹쳐지는가 — *Medium*
- [ ] **종이 템플릿** — 줄/모눈/점 배경이 바뀌는가 — *Low*
- [ ] **팜 리젝션(손바닥 무시)** ⚠️ — *태블릿 + 스타일러스 펜*이 있을 때만 진짜 검증 가능.
  - 펜으로 그리는 중 손바닥이 화면에 닿아도 손바닥 자국이 안 그려져야 함.
  - "펜 전용 모드"를 켰을 때 손가락으로는 안 그려지고 줌/이동만 되는가.
  - *주의: 손가락만 쓰는 폰에서는 이 항목을 정상 판정할 수 없음.*
- [ ] **줌/이동** — 두 손가락으로 확대·축소·이동이 되는가, 그 상태에서도 펜 그리기가 어긋나지 않는가 — *High*
- [ ] **자동 저장** ⚠️ — 그림을 그린 뒤 화면을 나갔다가 다시 들어오면 그림이 남아 있는가 — *High*
  - 저장은 "벡터 데이터(JSON) + 썸네일 PNG" 두 가지로 됩니다. 한쪽만 되고 한쪽이 안 될 수 있으니 둘 다 확인.

### 4-2. 스캔 이미지 첨삭 (scan_annotation) — ⚠️ 좌표 정합이 최대 난관

- [ ] **이미지 불러오기** — 갤러리에서 사진을 고르면 첨삭 화면에 뜨는가 — *Critical*
  - 문서 스캔(카메라로 종이 스캔) 기능은 현재 **꺼둔 상태**(실기기 전용 라이브러리라 임시 비활성). 갤러리 경로로 먼저 점검.
- [ ] **사진 위에 펜 주석** — 이미지 위에 빨강/파랑 펜으로 그어지는가 — *Critical*
- [ ] **작성자 색 구분** — 멘토와 학생의 펜 색이 자동으로 다르게 나오는가 — *Medium*
- [ ] **좌표 정합** ★★★ 가장 중요 ★★★ — *Critical*
  - 사진의 특정 글자 위에 동그라미를 쳤다고 할 때:
    - **확대했다가 원래대로 돌아와도** 동그라미가 같은 글자 위에 그대로 있는가?
    - **앱을 껐다 켜거나 다른 기기에서 열어도** 같은 위치에 있는가?
  - 이게 어긋나면(글자에서 동그라미가 떨어져 있으면) → 정규화 좌표(0~1 비율 저장) 로직 버그.
  - 의심 파일: `scan_annotation.dart` 의 `ScanCoordMapper` (normalize / denormalize), 그리고 `scan_annotation_canvas.dart` 의 이미지 종횡비 처리.
  - AI 도구에 넘길 때: "스캔 첨삭에서 확대 후 주석 위치가 어긋난다. 정규화 좌표 매핑을 점검해줘" 라고 구체적으로.
- [ ] **세로/가로 사진** — 세로로 긴 사진, 가로로 넓은 사진 모두에서 주석이 안 어긋나는가 — *High*
- [ ] **저장** — 나갔다 들어오면 주석이 남는가 (주석 JSON + 미리보기 PNG) — *High*

---

## 5. C단계 — 태블릿 UI 점검 (Task 4)

> 태블릿이 주 사용 환경이므로 별도로 점검합니다.

- [ ] **T-1.** 태블릿에서 하단 탭 대신 **좌측 세로 레일**로 바뀌는가 — *Medium*
- [ ] **T-2.** 필기 툴바가 태블릿에서 **측면 레일** 형태로, 폰에서 **하단 바** 형태로 나오는가 — *Medium*
- [ ] **T-3.** 넓은 화면에서 내용이 가운데로 적당히 모이는가(양옆 여백) — *Low*
- [ ] **T-4.** 가로 모드(landscape)로 돌렸을 때 레이아웃이 깨지지 않는가 — *Medium*
- [ ] **T-5.** 글자/버튼이 태블릿에서 너무 작거나 너무 크지 않은가 — *Low*

## 6. 알려진 "버그 아님" 목록 (스텁/미연동)

아래 항목은 **고장이 아니라, 일부러 비워둔 부분**입니다. QA에서 "안 됨"으로 잡지 말고, 이어서 채울 작업으로 분류하세요. (자세한 채우기 순서는 `docs/EXPO_FEATURE_REFERENCE.md` 참고)

- 커뮤니티 / 숏폼 / 멘토 검색 / 맞춤 의뢰 / 마이페이지 / 멘토 대시보드 / 정산 — **빈 화면(스텁)**. 안내 문구만 표시됨.
- **로그인·회원가입** — 현재는 데모용 역할 전환만 있고, 실제 Supabase 인증은 미연동.
- **결제(토스페이먼츠)** — 캐시 충전·구독 화면 UI는 있으나 실제 앱 결제 연동은 미정.
- **문서 카메라 스캔** — 라이브러리는 넣어뒀으나 실기기 전용이라 코드에서 임시로 꺼둠. 갤러리 불러오기로 대체 동작.
- **실제 데이터(질문방 목록 등)** — 지금은 예시 더미 데이터. Supabase 연결 후 실데이터로 교체 필요.

## 7. 권장 디버깅 순서 (요약)

복잡할 땐 이 순서만 기억하세요.

1. **빌드를 살린다** (A단계 C-1~C-3) — 빨간 에러 0개 만들기
2. **앱을 켜고 이동을 살린다** (B-1~B-6) — 역할 선택 → 탭 이동 → 노트/스캔 진입
3. **필기를 살린다** (4-1) — 그리기 → undo/redo → 자동저장
4. **스캔 좌표 정합을 잡는다** (4-2, 특히 ★ 항목) — 확대 후/재실행 후 위치 유지
5. **태블릿 레이아웃을 다듬는다** (5)
6. **스텁 화면을 채운다** (EXPO_FEATURE_REFERENCE.md 우선순위대로)
7. **Supabase·결제 실연동**
8. **마지막에 색·간격·문구**(Low) 한꺼번에 정리

## 8. 버그를 AI 도구에 넘기는 좋은 양식

발견한 문제를 아래 형식으로 적어서 그대로 붙여 넣으면 수정이 훨씬 빨라집니다.

```
[화면] 스캔 첨삭 화면
[기기] 갤럭시탭 (태블릿, 가로 모드)

[기대] 동그라미가 같은 글자 위에 그대로
[실제] 동그라미가 글자보다 오른쪽 아래로 밀려남
[심각도] Critical (핵심 기능)
[의심 파일] lib/features/scan_annotation/models/scan_annotation.dart 의 ScanCoordMapper
```
